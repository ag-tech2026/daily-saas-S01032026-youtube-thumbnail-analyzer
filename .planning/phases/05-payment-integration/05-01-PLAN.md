    3. Update `src/lib/auth.ts` — Add Polar plugin to BetterAuth config:
       - Import `polar`, `checkout`, `webhooks` from `@polar-sh/better-auth`
       - Import `polarClient` from `./polar`
       - Import `db` from `./db`, `{ eq, sql }` from `drizzle-orm`, `{ user, webhookEvents }` from `./schema`
       - Add `polar()` plugin to the existing `plugins` array (create the array if it doesn't exist — currently there is no plugins array)
       - Configure `polar({ client: polarClient, createCustomerOnSignUp: true, use: [...] })`
       - In `use` array, add `checkout()` with:
         - `products: [{ productId: process.env.POLAR_PRODUCT_ID!, slug: "credits-50" }]`
         - `successUrl: "/dashboard?purchase=success"` (per user decision: redirect to dashboard)
         - `authenticatedUsersOnly: true`
       - In `use` array, add `webhooks()` with:
         - `secret: process.env.POLAR_WEBHOOK_SECRET!`
         - `onOrderPaid: async (payload) => { ... }` handler that:
           a. Extracts `userId` from `payload.data.customer?.externalId` (note: the Polar SDK uses camelCase `externalId`, NOT snake_case — verify at runtime and fallback to `payload.data.customer?.external_id` if needed)
           b. Extracts `webhookId` from `payload.data.id` (the order ID, which is unique per order and safe for idempotency)
           c. If no userId, logs error and returns early
           d. Uses `db.transaction(async (tx) => { ... })`:
              - SELECT from `webhookEvents` WHERE id = webhookId, limit 1
              - If exists, log "already processed" and return (idempotent skip)
              - INSERT into `webhookEvents` with id, type "order.paid", processedAt
              - UPDATE `user` SET `credits = credits + 50` WHERE `id = userId` (use `sql` template for atomic increment)
           e. Logs success message

    4. Update `src/lib/auth-client.ts` — Add Polar client plugin:
       - Import `polarClient` from `@polar-sh/better-auth` (note: this is the CLIENT export `polarClient`, not the SDK `Polar` class)
       - Add `plugins: [polarClient()]` to `createAuthClient` config
       - The `authClient.checkout()` method will now be available

    5. Update `src/lib/schema.ts` — Add `webhookEvents` table:
       - Add `export const webhookEvents = pgTable("webhook_events", { ... })` with:
         - `id: text("id").primaryKey()` — Polar order ID
         - `type: text("type").notNull()` — event type e.g. "order.paid"
         - `processedAt: timestamp("processed_at").defaultNow().notNull()`
         - `payload: text("payload")` — optional, store JSON for debugging
       - Place it after the `analysis` table definition

    6. Push schema to database:
       `pnpm run db:push` (using db:push per project convention, not db:migrate)

    7. Update `env.example` — Add `POLAR_PRODUCT_ID`:
       - Add `POLAR_PRODUCT_ID=` under the existing Polar section (POLAR_WEBHOOK_SECRET and POLAR_ACCESS_TOKEN already exist)

    8. Run `pnpm run lint && pnpm run typecheck` to verify no errors.

    IMPORTANT: The BetterAuth catch-all route at `src/app/api/auth/[...all]/route.ts` already handles all auth routes including the new Polar webhook endpoint at `/api/auth/polar/webhooks` and checkout at `/api/auth/checkout/credits-50`. No changes needed to that file.
