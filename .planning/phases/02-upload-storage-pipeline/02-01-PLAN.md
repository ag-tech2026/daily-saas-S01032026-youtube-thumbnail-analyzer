<task type="auto">
  <name>Task 2: Create POST /api/upload endpoint with atomic credit deduction</name>
  <files>src/app/api/upload/route.ts</files>
  <action>
Create `src/app/api/upload/route.ts` with a POST handler. Follow the same auth pattern as `src/app/api/chat/route.ts`.

**Authentication:**
- Get session via `auth.api.getSession({ headers: await headers() })` — return 401 if no session.

**Credit check + atomic deduction:**
- Use Drizzle's `sql` template literal for an atomic UPDATE that decrements credits by 1 ONLY if credits > 0:
  ```ts
  const result = await db.update(user)
    .set({ credits: sql`${user.credits} - 1` })
    .where(and(eq(user.id, session.user.id), gt(user.credits, 0)))
    .returning({ credits: user.credits });
  ```
- If `result.length === 0`, the user had 0 credits — return 402 with `{ error: "Insufficient credits" }`.
- Import `eq`, `gt`, `and`, `sql` from drizzle-orm; import `user`, `analysis` from `@/lib/schema`.

**File extraction and validation:**
- Parse the request as FormData: `const formData = await req.formData()`
- Get file from `formData.get("file")` — must be a `File` instance, else return 400.
- Validate on server side:
  - File size > 10MB (10 * 1024 * 1024 bytes) -> return 400 `{ error: "File too large. Maximum size is 10MB" }`
  - File type not in ["image/jpeg", "image/png"] -> return 400 `{ error: "Only PNG and JPG images are allowed" }`
- Convert File to Buffer: `Buffer.from(await file.arrayBuffer())`

**Create analysis record:**
- Generate UUID: `const analysisId = crypto.randomUUID()`
- Insert into analysis table with status "pending", userId from session, imageUrl null initially.

**Upload to storage:**
- Use `upload()` from `@/lib/storage` with folder "screenshots" and filename `${analysisId}-${sanitizedOriginalName}`.
- Use `sanitizeFilename` from `@/lib/storage` for the original filename.

**Update analysis record:**
- After successful upload, update the analysis record to set `imageUrl` to the storage URL and `status` to "uploaded".

**Return response:**
- Return 200 with `{ analysisId, imageUrl: storageResult.url, credits: result[0].credits }`.

**Error handling:**
- If storage upload fails, leave analysis record as "pending" (for retry/refund in later phases). Return 500 with `{ error: "Upload failed" }`.
- Wrap the entire flow in try/catch. On unexpected errors, return 500.

**Do NOT use a database transaction wrapping everything** — the credit deduction is atomic via the WHERE clause, and if upload fails the analysis record stays "pending" which is the correct recovery state.
