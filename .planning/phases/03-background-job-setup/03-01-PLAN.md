<objective>
Install Inngest, create type-safe client with event schemas, define the process-analysis background function with step.run() for database status updates, wire the serve handler, and trigger events from the upload API.

Purpose: Enables asynchronous processing of poker screenshot uploads. When a user uploads a screenshot, an Inngest event fires to process the analysis in the background, updating status from "uploaded" to "processing". This lays the foundation for Phase 4's AI vision analysis to plug into the existing step-based workflow.

Output: Working Inngest integration with typed events, a process-analysis function that updates DB status, serve handler at /api/inngest, and upload API triggering events on successful upload.
</objective>

2. After the existing `db.update(analysis).set({ imageUrl: storageResult.url, status: "uploaded" })` call (around line 103-109), add the inngest.send() call wrapped in try/catch so event delivery failure does not fail the upload:

```typescript
// Trigger background job for analysis processing
try {
  await inngest.send({
    name: "analysis/upload.completed",
    data: {
      analysisId,
      userId: session.user.id,
      imageUrl: storageResult.url,
    },
  });
} catch (inngestError) {
  // Log but don't fail the upload - analysis record exists for retry
  console.error("Failed to trigger analysis job:", inngestError);
}
```

3. Place the inngest.send() AFTER the database update succeeds but BEFORE the success response is returned. This ensures the analysis record exists with imageUrl before the background function tries to read it.

4. The try/catch ensures the upload succeeds even if Inngest Dev Server is not running or event delivery fails. The analysis stays in "uploaded" status and can be reprocessed later.
