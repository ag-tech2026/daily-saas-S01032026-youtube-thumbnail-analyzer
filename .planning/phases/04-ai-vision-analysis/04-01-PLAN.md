**Create `src/lib/analysis-schema.ts`:**

Define the structured output schema for poker hand analysis using Zod. This schema is what the vision model must conform to via `Output.object()`.

```typescript
import { z } from "zod";

const streetAnalysisSchema = z.object({
  street: z.enum(["preflop", "flop", "turn", "river"]).describe("Which street this analysis covers"),
  heroAction: z.string().describe("What Hero did on this street (e.g., 'raised to 3BB', 'called', 'checked', 'folded')"),
  gtoVerdict: z.enum(["correct", "acceptable", "incorrect"]).describe("GTO assessment of Hero's action"),
  explanation: z.string().describe("Beginner-friendly explanation of why this action is correct/incorrect. Use 'Hero' third person voice. Explain poker terms in context. If Hero deviated from GTO, explain what the GTO play is and why."),
  rangeNote: z.string().optional().describe("Simplified range guidance in categories (e.g., 'strong top pairs and better', 'suited broadways'). Never list specific hands."),
  evNote: z.string().optional().describe("Directional EV note only (e.g., 'this call loses money long-term', 'this bet is profitable'). Never use numerical EV."),
});

export const analysisSchema = z.object({
  streets: z.array(streetAnalysisSchema)
    .min(1)
    .describe("Analysis for each street that occurred in the hand. If hand ended preflop, array has 1 item. Full hand to river has 4 items. Only include streets that actually occurred."),
  overallVerdict: z.enum(["GTO-Compliant", "Needs Improvement"])
    .describe("Binary verdict for the entire hand"),
  verdictSummary: z.string()
    .describe("1-2 sentence summary explaining the verdict. Focus on the biggest mistake or best play."),
  keyTakeaway: z.string()
    .describe("The single most important lesson from this hand. What should Hero remember and apply next time?"),
});

export type AnalysisResult = z.infer<typeof analysisSchema>;
```

Use `.describe()` on EVERY field to guide the vision model. Use enums for constrained fields to prevent case/format issues.

9. **Verdict:** Binary "GTO-Compliant" or "Needs Improvement" with summary
10. **Key takeaway:** Single biggest mistake or best play
11. **Order:** Street-by-street details first, building to verdict (tells a story)
12. **Poker sites:** Must handle PokerStars, GGPoker, and 888poker screenshots
