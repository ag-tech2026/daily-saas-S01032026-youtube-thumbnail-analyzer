<task type="auto">
  <name>Task 1: Create analysis API routes (list and single)</name>
  <files>src/app/api/analyses/route.ts, src/app/api/analyses/[id]/route.ts</files>
  <action>
Create two API route handlers:

**1. `src/app/api/analyses/route.ts` — GET list endpoint:**
- Import `auth` from `@/lib/auth`, `db` from `@/lib/db`, `analysis` from `@/lib/schema`, `headers` from `next/headers`, `eq` and `desc` from `drizzle-orm`.
- Check session with `auth.api.getSession({ headers: await headers() })`. Return 401 JSON if no session.
- Query: `db.select().from(analysis).where(eq(analysis.userId, session.user.id)).orderBy(desc(analysis.createdAt)).limit(50)`.
- Return JSON array response with `Content-Type: application/json`.

**2. `src/app/api/analyses/[id]/route.ts` — GET single endpoint:**
- Same auth check pattern.
- Await params: `const { id } = await params` (Next.js 16 async params — type is `Promise<{ id: string }>`).
- Query with IDOR protection: `db.select().from(analysis).where(and(eq(analysis.id, id), eq(analysis.userId, session.user.id))).limit(1)`.
- Return 404 JSON if no rows (do NOT return 403 — avoids disclosing existence).
- Return the single analysis row as JSON.
  </action>
  <verify>Run `pnpm run typecheck` — both routes must compile without errors. Run `pnpm run lint` — no lint errors.</verify>
  <done>GET /api/analyses returns JSON array of user's analyses sorted newest-first. GET /api/analyses/[id] returns single analysis with userId ownership check, 404 if not found or not owned.</done>
</task>

**3. Modify `src/app/dashboard/page.tsx` to add history section:**
- Add imports: `Badge` from `@/components/ui/badge`, `Clock` and `CheckCircle2` and `XCircle` and `AlertCircle` from `lucide-react`, `useState` from `react`.
- Add a `AnalysisResult` type import from `@/lib/analysis-schema`.
- Define a type for the list items: `type AnalysisListItem = { id: string; status: string; result: string | null; createdAt: string; imageUrl: string | null; }`.
- Add state: `const [analyses, setAnalyses] = useState<AnalysisListItem[]>([])` and `const [historyLoading, setHistoryLoading] = useState(true)`.
- Add a `useEffect` that fetches `/api/analyses` on mount (when `session` exists), sets `analyses` state, and sets `historyLoading` to false. Depend on `[session]`.
- Define a helper function `getStatusBadge(status: string)` that returns `{ label: string; variant: "default" | "secondary" | "destructive" | "outline" }`:
  - `"complete"` -> `{ label: "Complete", variant: "default" }`
  - `"failed"` -> `{ label: "Failed", variant: "destructive" }`
  - `"processing"` -> `{ label: "Processing", variant: "secondary" }`
  - `"pending"` or `"uploaded"` -> `{ label: "Pending", variant: "outline" }`
  - default -> `{ label: status, variant: "outline" }`
- Define a helper function `getOverallVerdict(item: AnalysisListItem)` that returns `string | null`:
  - If `item.status !== "complete"` or `!item.result`, return `null`.
  - Try `JSON.parse(item.result)` and return `parsed.overallVerdict`. Catch and return `null`.
- After the existing grid `</div>`, add a new section:
  - `<h2>` with "Recent Analyses" title and a history icon.
  - If `historyLoading`, show 3 skeleton cards (use Skeleton component from `@/components/ui/skeleton`).
  - If `!historyLoading && analyses.length === 0`, show an empty state message: "No analyses yet. Upload a poker screenshot to get started!" with a link to `/upload`.
  - If analyses exist, render a list of clickable cards. Each card is wrapped in a `<Link href={/analysis/${item.id}}>` and shows:
    - Status badge (using `getStatusBadge`).
    - Date formatted with `new Date(item.createdAt).toLocaleDateString("en-US", { month: "short", day: "numeric", year: "numeric" })`.
    - Overall verdict text if complete (from `getOverallVerdict`), or a brief status description if not complete.
    - A right-arrow indicator showing it's clickable.
  - Limit displayed items to the most recent 10 with a "View all" option if more exist (but since API already returns 50, just show all).
  </action>
