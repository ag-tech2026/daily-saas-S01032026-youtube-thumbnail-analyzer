Create `src/app/analysis/[id]/page.tsx` as a Client Component (`"use client"`).

**Component structure:**
- The page receives `params: Promise<{ id: string }>` (Next.js 16 async params).
- Since this is a Client Component, use `React.use(params)` to unwrap the promise (or unwrap in a useEffect). The simplest approach: accept `params` as a prop, call `React.use(params)` at the top level to extract `id`.

**State management:**
- `data`: the analysis row `{ id, status, result, imageUrl, createdAt, updatedAt } | null`
- `loading`: boolean, starts true
- `error`: string | null

**Polling logic (reuse exact pattern from dashboard/page.tsx):**
- `useRef<NodeJS.Timeout | null>(null)` for poll interval reference.
- `useEffect` depending on `[id]`:
  1. Define async `fetchAnalysis()` that calls `fetch(/api/analyses/${id})`.
  2. On success, parse JSON and set `data`. Set `loading` to false.
  3. If response is 404, set `error` to "Analysis not found" and clear interval.
  4. Inside the fetch callback, after setting data, check: if `status === "complete"` or `status === "failed"`, call `clearInterval(pollRef.current)`.
  5. Call `fetchAnalysis()` immediately (initial load).
  6. Start `setInterval(fetchAnalysis, 3000)` and store in `pollRef.current`.
  7. Return cleanup function that calls `clearInterval(pollRef.current)`.

**Rendering states:**

1. **Loading (initial):** Show a page skeleton with Card-shaped placeholders using Skeleton component. Include a back link to `/dashboard`.

2. **Error (404):** Show "Analysis not found" with a link back to dashboard.

3. **Pending / Uploaded / Processing:** Show a waiting state:
   - Back link to `/dashboard` (use `<Link>`).
   - Card with Spinner component and text: "Analyzing your hand..." for processing, "Waiting to start..." for pending/uploaded.
   - Status badge using the same `getStatusBadge` helper as Plan 01 (define locally or import — define locally to keep the file self-contained).
   - Show the uploaded image if `imageUrl` exists, using `<img>` tag (not Next.js Image) with `className="max-w-sm rounded-lg border"`. Alt text: "Uploaded poker screenshot".
   - Show `createdAt` formatted date.

4. **Failed:** Show an error state:
   - Back link to `/dashboard`.
   - Card with XCircle icon, red accent.
   - Text: "Analysis Failed" heading, "We couldn't analyze this hand. Your credit has been refunded." body.
   - Button linking to `/upload`: "Try Another Hand".
   - Show the uploaded image if available.

5. **Complete:** Full result display. Parse `JSON.parse(data.result)` as `AnalysisResult` type from `@/lib/analysis-schema`.
   - Back link to `/dashboard`.
   - **Header section:** Overall verdict badge (green for "GTO-Compliant", yellow/amber for "Needs Improvement"), formatted date.
   - **Verdict summary:** Card with `verdictSummary` text.
   - **Key takeaway:** Card with lightbulb icon and `keyTakeaway` text, visually distinct (e.g., blue/primary accent border).
   - **Street-by-street analysis:** For each item in `streets` array, render a Card:
     - Street name capitalized as heading (e.g., "Preflop", "Flop", "Turn", "River").
     - "Hero's Action:" with `heroAction` text.
     - GTO verdict badge with color coding:
       - `"correct"` -> green text/badge: "Correct"
       - `"acceptable"` -> yellow text/badge: "Acceptable"
       - `"incorrect"` -> red text/badge: "Incorrect"
     - `explanation` text as the main body.
     - If `rangeNote` exists, show in a subsection with "Range" label.
     - If `evNote` exists, show in a subsection with "EV Impact" label.
   - **Screenshot:** Show uploaded image at bottom with `<img>` tag if `imageUrl` exists.

**Styling guidelines:**
