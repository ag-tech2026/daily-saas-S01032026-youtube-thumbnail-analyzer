# Requirements: Poker Hand Review AI

**Defined:** 2026-02-13
**Core Value:** Users upload a single poker screenshot and get a clear, actionable GTO analysis that helps them understand what they did right or wrong

## v1 Requirements

### Authentication

- [ ] **AUTH-01**: User can sign up with email and password
- [ ] **AUTH-02**: User can log in with Google OAuth
- [ ] **AUTH-03**: User receives 3 free credits on first signup (no CC required)
- [ ] **AUTH-04**: User session persists across browser refresh

### Hand Analysis

- [ ] **ANLYS-01**: User can upload a poker screenshot via drag-and-drop or file picker
- [ ] **ANLYS-02**: System validates image file type (PNG/JPG) and size (max 10MB) before upload
- [ ] **ANLYS-03**: Screenshot is stored in Vercel Blob with reference in database
- [ ] **ANLYS-04**: Upload triggers Inngest background job for AI analysis
- [ ] **ANLYS-05**: AI analyzes screenshot using OpenRouter vision model with GTO-focused prompt
- [ ] **ANLYS-06**: Analysis evaluates each player action (preflop, flop, turn, river) with GTO verdict (correct/incorrect/acceptable)
- [ ] **ANLYS-07**: Each action includes explanation of what GTO recommends and why
- [ ] **ANLYS-08**: Analysis includes range analysis (what hands should be in range at key decision points)
- [ ] **ANLYS-09**: Analysis includes EV notes (expected value impact of key decisions)
- [ ] **ANLYS-10**: Analysis includes overall hand verdict (GTO-compliant or not, with beginner-friendly summary)
- [ ] **ANLYS-11**: Analysis results are saved to database with structured format

### Credits & Payments

- [ ] **CRED-01**: 1 credit is deducted per analysis before processing starts
- [ ] **CRED-02**: User can view remaining credit balance in header/dashboard
- [ ] **CRED-03**: User cannot upload if credit balance is zero (shown upgrade prompt)
- [ ] **CRED-04**: Credit is refunded if analysis fails
- [ ] **CRED-05**: User can purchase 50 credits for $9 via Polar checkout
- [ ] **CRED-06**: Credits are added after successful Polar webhook confirmation

### History & Dashboard

- [ ] **DASH-01**: User can view list of past hand analyses with date and verdict summary
- [ ] **DASH-02**: User can click on a past analysis to view the full detail page
- [ ] **DASH-03**: Analysis shows processing status (pending/processing/complete/failed)
- [ ] **DASH-04**: Dashboard displays current credit balance

### Landing Page

- [ ] **LAND-01**: Landing page has hero section with clear value proposition
- [ ] **LAND-02**: Landing page shows demo/example analysis
- [ ] **LAND-03**: Landing page displays pricing (free 3 credits + $9 pro pack)
- [ ] **LAND-04**: Landing page has sign-up CTA

## v2 Requirements

### Enhanced Analysis

- **ANLYS-V2-01**: Visual 13x13 range grid display (color-coded by action)
- **ANLYS-V2-02**: Position-aware advice (BTN vs BB strategy differences)
- **ANLYS-V2-03**: Confidence scoring on each analysis
- **ANLYS-V2-04**: Poker site auto-detection (PokerStars, GGPoker, 888poker)

### Dashboard Enhancements

- **DASH-V2-01**: Auto-refresh polling while analysis is processing
- **DASH-V2-02**: Common mistakes summary across all past analyses
- **DASH-V2-03**: Win rate trends over time

### Additional Plans

- **CRED-V2-01**: Multiple credit pack tiers
- **CRED-V2-02**: Monthly subscription option

## Out of Scope

| Feature | Reason |
|---------|--------|
| Real-time coaching overlay | Against poker site TOS, legal issues |
| Training courses | Scope creep, different product entirely |
| Hand replayer | High complexity, not core to analysis value |
| Social features (sharing, comments) | Adds moderation burden, not core value |
| Poker HUD integration | Against site TOS, requires desktop app |
| Multi-street multi-screenshot analysis | 3-4x more complex UX and cost, v1 is single screenshot |
| Tournament ICM calculations | Niche audience, focus on cash games first |
| Mobile app | Web-first, mobile responsive is sufficient |
| Hand history text parsing | Image-only for v1, simpler UX |
